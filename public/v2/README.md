# NAS CodeWorks — دليل التشغيل والنشر

## الملفات

| الملف                | الوظيفة                                               |
| -------------------- | ----------------------------------------------------- |
| `index.html`         | الموقع العام (للزوار)                                 |
| `render.js`          | محرك بناء الموقع + السلايد شو                         |
| `admin.html`         | لوحة التحكم                                           |
| `admin.js`           | منطق لوحة التحكم                                      |
| `content.json`       | المحتوى الافتراضي (المصدر الأساسي)                    |
| `content-default.js` | نسخة احتياطية مضمّنة من المحتوى (تعمل حتى بدون سيرفر) |

> **مهم:** كل الملفات تبقى مع بعض بنفس المجلد. لا تفصلها.

---

## التشغيل المحلي (للتجربة)

افتح المجلد بسيرفر بسيط — **لا تفتح index.html مباشرة بنقرة مزدوجة** (بعض المتصفحات تمنع `fetch` على `file://`):

```bash
cd مجلد-الموقع
python3 -m http.server 8080
```

- الموقع: `http://localhost:8080/index.html`
- لوحة التحكم: `http://localhost:8080/admin.html`
- رمز الدخول الافتراضي: `nascw2026` (غيّره من تبويب الإعدادات داخل اللوحة)

## كيف يشتغل المحتوى (المرحلة الحالية — localStorage)

1. تفتح `admin.html`، تعدّل النصوص/الأسعار/العروض/الاتصال، وتضيف صور البرامج.
2. تضغط **حفظ** → كل شي ينحفظ داخل متصفحك (`localStorage`).
3. تضغط **معاينة الموقع** → تشوف النتيجة فوراً.

> **انتبه:** تعديلات `localStorage` تبقى **بمتصفحك أنت فقط**. الزوار يشوفون المحتوى من `content.json` (وملف `content-default.js`).

### لنشر تعديلاتك للزوار (بدون سيرفر / بدون Backend):

1. عدّل بلوحة التحكم واحفظ.
2. اضغط **تصدير JSON** → ينزل ملف `content.json` جديد فيه تعديلاتك.
3. **حدّث ملفين** في مجلد الموقع بالمحتوى الجديد:
    - استبدل `content.json` بالملف المُصدَّر.
    - حدّث `content-default.js`: محتواه هو نفس JSON بس مسبوق بـ `window.__DEFAULT_CONTENT__=` ومتبوع بـ `;` — أسهل طريقة: انسخ محتوى `content.json` كاملاً بين هذين.
4. ارفع المجلد كله لاستضافة static (Netlify / Cloudflare Pages / GitHub Pages) — مجاناً.

الآن كل الزوار يشوفون نسختك المحدّثة.

---

## نشر سريع مجاني (موصى به)

**Cloudflare Pages أو Netlify:**

1. اسحب مجلد الموقع كاملاً لصفحة الرفع، أو اربطه بمستودع GitHub.
2. لا حاجة لأي build — الموقع static.
3. تحصل على رابط فوري (مثل `nascodeworks.pages.dev`)، وتقدر تربط دومين خاص لاحقاً.

## `/admin` برمز سري — المرحلة الثانية (لاحقاً)

الرمز الحالي بلوحة التحكم **يحمي الواجهة فقط** (يُخزَّن بالمتصفح). أي شخص يفتح كود الصفحة يقدر يشوفه — مو حماية فعلية.

للحماية الحقيقية على مسار `/admin` بعد النشر، تحتاج **Backend**:

### الخيار الأبسط (Cloudflare):

- **Cloudflare Pages** للموقع + **Cloudflare Access** يحمي مسار `/admin` بإيميل/كلمة مرور حقيقية (مجاني حتى 50 مستخدم).
- المحتوى ينحفظ بـ **Cloudflare KV** (تخزين مركزي) بدل localStorage.

### ما الذي يتغيّر بالكود:

- **لا نعيد بناء اللوحة.** نفس `admin.js` و`admin.html`.
- نبدّل **3 دوال فقط**: `save()`, `loadAll()`, و`loadContent()` — بدل localStorage، تنادي API بسيط (مثلاً `/api/content` GET/POST محمي).
- صمّمنا طبقة التخزين معزولة من البداية تحديداً لهذا الانتقال السلس.

قُل لي وقت تكون جاهز للمرحلة الثانية، وأبنيها.

## ملاحظات تقنية

- **الصور:** تُضغط تلقائياً عند الرفع (حد أقصى عرض 1600px، JPEG ~82%) قبل التخزين، لتقليل الحجم.
- **حد localStorage:** ~5MB لكل دومين. مع كثرة الصور قد يمتلئ — استعمل **تصدير JSON** للنسخ الاحتياطي دورياً، وفضّل روابط الصور الخارجية إذا الصور كثيرة جداً.
- **السلايد شو:** يدعم لمس (swipe)، أسهم الكيبورد، نقر الثمبنيل/النقاط، تكبير بالنقر (zoom)، وعدّاد — RTL بالكامل.
- **الواتساب:** كل أزرار الموقع تستعمل `contact.waNumber` — غيّر الرقم من مكان واحد بلوحة التحكم وينطبق على الكل.
